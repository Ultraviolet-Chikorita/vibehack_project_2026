# Settld

Settld is a dispute-evidence platform for merchants. It reads mailbox activity, groups related messages into order-level evidence vaults, detects disputes, and prepares response packs for human review.

## Repository layout

- `README.md` — repository overview and working notes
- `codeplainfiles/READMEplain.md` — Codeplain-specific documentation and module map
- `codeplainfiles/*.plain` — source-of-truth spec modules for the dispute engine

## What the specs cover

- mailbox filtering and commerce classification
- structured event extraction
- order evidence vault creation and linking
- evidence scoring and dispute detection
- evidence pack generation
- billing and metering
- pipeline orchestration and sync handling

## Working with the specs

The `.plain` files are the canonical source for the engine generated by Codeplain.

```bash
plain2code codeplainfiles/app.plain
```

`app.plain` imports the full module set and should be used for the integrated render.

## Documentation conventions

- Keep shared concepts in `dispute-engine-core.plain`.
- Put feature-specific definitions beside the feature they support.
- Keep acceptance tests close to the functional requirement they verify.
- Prefer concise, direct prose over marketing copy or placeholder comments.
